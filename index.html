排序算法之快速排序算法

#排序算法之快速排序算法（升序）

##基本原理
运用分治的思想；取数组的第一个数为关键数，根据这个数将数组分成两个部分，使得第一部分的数全部小于等于该数，而第二部分的数全部大于等于该数；重复上面的步骤直到排序完成。

##实现步骤
待排序数组array的长度为N，令i=0,j=N-1;令key=array[0];

  1. 判断array[j]是否小于key，若是，则交换array[j]与array[i]的值，否则j–,并重复步骤1;
  2. 判断array[i]是否大于key，若是，则交换array[j]与array[i]的值，否则i++，并重复步骤2；
  3. 判断i是否小于j，若是，则重复步骤1、2；否则，数组被分成0到i-1和i+1到N-1两个部分，并将两部分用同样的步骤进行处理，直到分成后的部分的长度小于等于1为止

##代码实现

/**
* 快速排序算法
* <p>Description:对array数组中从下标为start到下标为end的部分的进行处理</p> 
* @param array 待排序数组
* @param start 开始下标
*@param end 结束下标
*/
public void QuickSort(int[] array,int start,int end){
    if(start>=end)//结束条件
        return ;
    int key=array[start];
    int i=start;
    int j=end;
    while(i<j){

       while(j>=0&&array[j]>key){//从j向前查找小于key的数
           j--;
       }
       if(j>i){//交换array[i]与array[j]
         int temp=array[i];
         array[i]=array[j];
         array[j]=temp;
       }
       while(i<=end&&array[i]<key){//从i开始向后查找大于key的数
           i++;
       }
       if(i<j){//交换array[i]与array[j]
       int temp=array[i];
           array[i]=array[j];
           array[j]=temp;
       }
    }
  QuickSort(array,start,i-1);//递归第一部分    
  QuickSort(array,i+1,end);    //递归第二部分

}

每天几道编程题总结(1)

题1：汇编语言中有一种移位指令叫做循环左移（ROL），现在有个简单的任务，就是用字符串模拟这个指令的运算结果。对于一个给定的字符序列S，请你把其循环左移K位后的序列输出。例如，字符序列S=”abcXYZdef”,要求输出循环左移3位后的结果，即“XYZdefabc”。是不是很简单？OK，搞定它！（来源：牛客网《剑指offer》）
解法一：该解法是做的时候首先想到的，每次左移一次都进行一个循环，时间复杂度为O(N*N)
public String LeftRotateString(String str, int n) {
if (str.isEmpty())
return “”;
char[] cstr = str.toCharArray();
for (int i = 0; i < n % cstr.length; i++) {
char temp = cstr[0];
for (int j = 0; j < cstr.length - 1; j++) {
cstr[j] = cstr[j + 1];
}
cstr[cstr.length - 1] = temp;
}
String s = new String(cstr);
return s;
}
解法二：”abcdefg”扩展为”abcdefgabcdefg”,截取n到n+len的部分即为所求字符串
public String LeftRotateString(String str, int n) {
int len = str.length();
if (len == 0)
return “”;
n = n % len;
str += str;
return str.substring(n, len + n);
}
解法三：分割法，将待左移的字符串”abcdefg”分割成两部分，需要移到末尾的为一部分，剩余的为另一部分，如，n=2,第一部分为”ab”,第二部分为”cdefg”,将第二部分加上第一部分就得到循环左移2位后的结果”cdefgab”
public String LeftRotateString(String str,int n) {
int len=str.length();
if(len==0)
return “”;
n=n%len;
String n1=str.substring(0,n);
String n2=str.substring(n,len);
return n2+n1;
}

三级网络技术备考知识点整理(7)

一、路由器工作原理
1、基本功能
（1）路由选择：依据目的IP地址的网络地址，通过路由选择算法确定一条从源结点到目的结点的最佳路由
A、核心：下一跳路由器的IP地址（路由表）
B、实现方法：使用路由选择算法，根据实际网络连接情况和网络性能，建立网络拓扑结构图，以建立路由选择和转发的基础。同时路由选择算法根据各自的判断原则（带宽、时延、负载、路由器跳数等因素），为网络上的路由产生一个权值。一般来说，权值愈小，路由愈佳。然后将最佳路由信息保存在一个路由表中。当网络拓扑发生变化时，路由选择协议会重新计算最佳路由，并更新路由表
（2）分组转发：按照路由选择所指出的路由将数据分组从源结点转发到目的结点
A、实现方法：接收到一个数据分组时，查看分组头中的目的IP地址的网络部分查询路由表。如果表中给出的是到达目的网络地址的下一跳路由器的IP地址，则按路由表给出的路径将数据分组转发到下一跳路由器，由下一跳路由器继续转发，直到将数据分组转发到目的端；如果目的网络与路由器的一个端口直接相连，那么对应于目的网络地址的路由表表项给出的是目的端口，这种情况下，路由器将数据分组直接发送给目的端口；如果路由表中既没有下一跳路由地址又没有目的端口时，将数据分组转发给缺省路由，由该路由所连的路由器继续转发到目的端，如果仍没有找到该分组的网络的路由信息，则将该分组丢弃。
2、路由表
（1）基本结构：有目的网络地址及其所对应的目的端口或下一跳路由地址和缺省路由
（2）管理距离：衡量路由器中给定的路由信息源的“可信度”。管理距离越小，可信度越高。
（3）权值：由带宽、时延、负载、跳数、可靠性、滴答数、花费等决定，权值愈小，路由愈佳
eg:[110/22]——>该条路由信息源的管理距离为110，权值为22

##交换机及其配置##
一、局域交换机
1、基本概念及特性：工作在数据链路层的网络设备，基于MAC地址识别，完成转发数据帧功能的一种网络连接设备。具有独占传输通道、独享信道带宽，同时允许多对站点进行通信、总带宽等于所有端口带宽之和的特性。
2、功能：
（1）建立和维护一个表示MAC地址与交换机端口对应关系的交换表
（2）在发送结点和接收结点之间建立一个虚连接；
（3）完成数据帧的转发或过滤
3、工作原理
通过一种自学习的方法，自动地建立和维护一个记录着目的MAC地址与设备端口映射关系的地址查询表
二、交换表的建立与维护
1、交换表的内容：目的MAC地址及其所对应的交换机端口号和所在的虚拟子网（VLAN ID）
2、交换表的建立：
3、交换表的维护：交换表须定时刷新，将长时间不用的表项删除，将新的表项存入。（每次储存一个地址表项时都被盖上一个时间戳；每当交换表中的一个地址被参考该表项会被盖上新的时间戳）
4、交换表的保存：交换表保存在可编址内容存储器（CAM）
5、显示交换表命令的使用方法
（1）大中型交换机：show cam dynamic
(2) 小型交换机：show mac-address-table
6、交换机工作原理实例
三、交换机的交换结构
1、软件执行交换结构
借助CPU和RAM硬件环境，有特定的软件来完成数据帧交换的一种交换结构；具有结构灵活、交换速度慢、交换机堆叠困难、交换机端口较多导致交换机性能下降的特点
2、矩阵交换结构
交换功能全部由硬件完成，有输入、输出、交换矩阵和控制处理四部分组成。具有交换速度快、延时小、结构紧凑、矩阵交换的实现相对较简单、不易扩展、不利于交换机性能的监控和运行管理的特点
3、总线交换结构
交换机背板上有一条公用总线系统，数据帧的交换都在总线上完成。具有性能好、便于堆叠扩展、易实现帧的广播、易监控管理、易实现多个输入对一个输出的帧传送的特点，但它对总线的带宽要求比较高，一般总线宽应为端口带宽的综和
4、共享存储器交换结构
四、交换机的交换模式
1、分类：存储转发、快速转发和碎片丢弃三种交换模式
2、快速转发交换模式：在交换机接收了帧的前14个字节，即收到6个字节的目的地址后便立即转发数据帧。仅将帧中的目的MAC地址部分拷贝到缓冲区中，而不缓存整个数据帧；优点是端口交换时间短、延时小、交换速度快；缺点是：不能提供检错纠错、速度匹配和流量控制等功能，交换机可靠性差
3、碎片丢弃交换模式：在开始转发数据帧前，先过滤掉造成大部分数据报错误的冲突碎片；先检查数据包的长度是否够64字节（512bit）,如果帧的长度小于64字节，则被视为碎片，交换机直接将该帧丢弃。优点是在开始转发数据帧之前就过滤掉了冲突碎片，确保了碎片不通过网络传播，提高了网络的传输速率和宽带利用率。
4、存储转发交换模式：在开始转发数据帧之前先接收整个数据帧，并把整个帧保存在缓冲区。特点是交换延时比较大，交换速度和对比比较慢，但具有链路差错校验功能、可靠性高、支持不同速率的端口等优点
五、交换机的分类
1、按所支持局域网标准分类：以太网交换机、FDDI交换机、ATM交换机和令牌环交换机
2、按架构分类（单台交换机、堆叠交换机、箱体模块化交换机）
（1）堆叠交换机：堆叠带宽为2~10Gbps,数量一般为6到8个；某些可达16个
（2）箱体模块化交换机：具有最好的性能、最好的灵活性和很好的安全性及可扩展性
3、按交换机工作在OSI参考模型的层次分类
（1）二层交换机（数据链路层）：仅能依据数据帧中的目的MAC地址进行数据帧的线速交换
（2）三层交换机（网络层）：具有路由功能的二层交换机，依据数据包中的目的IP地址，进行路径选择和快速的数据包交换；可以实现不同逻辑子网、不同VLAN之间的通信。
六、虚拟局域网VLAN技术
1、基本概念：一个网络设备或用户的逻辑组，该逻辑组是一个独立的逻辑网络、单一的广播域，该逻辑组设定不受实际交换机区段限制，也不受用户所在的物理地址和物理网段的限制；简化了网络的物理结构、提高了网络的易管理性和安全性，提高了网络的性能
2、技术特性：
（1）VLAN工作在数据链路层
（2）每个VLAN都是一个独立的逻辑网端，一个独立的广播域
（3）每个VLAN又是一个独立的逻辑网络，有唯一的子网号
3、标识
通常用VLAN ID和VLAN name标识。802.1Q中，VLAN ID用12位表示，支持4096个VLAN，1~1005是标准范围，1025~4096是扩展范围
4、VLAN Trunk（虚拟局域网中继技术）
交换机与交换机之间、交换机与路由器之间存在一条物理链路，而这一条物理链路上要传输多个VLAN信息的技术；其标准机制是帧标签
5、划分VLAN的方法
（1）基于端口划分VLAN：按交换机端口定义VLAN成员，每个交换机端口属于一个VLAN；简单有效、安全、易管理
（2）基于MAC地址：按每个连接到交换机设备的MAC地址定义VLAN成员；在新增用户或用户移动时减少配线间的管理工作
（3）基于第三层协议类型或地址：有利于组成基于应用的VLAN
七、生成树协议STP（802.1d）
1、主要功能：在保证网络中没有协议的基础上，允许在第二层链路中提供冗余路径，以保证网络可靠、稳定的运行
2、基本工作原理：通过在交换机之间传递网桥协议数据单元（BPDU），并用生成树算法（STA）对其进行计算。根据BPDU提供的参数和STA的计算，STP先确定一个根（根网桥），然后确定交换机冗余链路端口的工作状态，让一些端口进入阻塞工作模式，另一些端口进入转发工作模式。其中被阻塞的端口仍然是一个激活的端口，但它只能接收和读取BPDU，不能接收和转发数据。用改变冗余端口的工作状态来阻断网络中的部分冗余路径，使其成为备份路径，以保证在任何两个终端站点之间，只存在一条激活的路径，避免了回路的产生。STP还计算从根到二层网络中所有交换机的最佳路径，并建立一个无环路的树状结构网络
3、关于BPDU：携带了实现生成树算法的有关信息，包括Boot ID、Root Path Cost、Bridge ID、Port ID、Hello time、Max Age等

##局域网技术##
一、局域网组网的基础知识
1、局域网（LAN）的定义：分布在相对较小区域范围内的计算机网络
2、局域网标准：IEEE 802标准包括局域网参考模型与各层协议
3、交换式局域网：核心部件是交换机，而交换机采用快捷交换和存储转发交换两种转发技术
4、虚拟局域网（VLAN）：以交换技术为基础，802.1Q为标准
组网定义方法：（1）用交换机端口定义虚拟局域网；（2）MAC地址定义虚拟局域网（3）利用网络层协议定义虚拟局域网；（4）基于IP广播组的虚拟局域网
二、综合布线（PDS）
１、定义：模块化的、灵活性较高的建筑群间或建筑内的信息传输通道；由不同系列和规模的部件组成，其中包括传输介质、相关连接硬件（配线器、连接器、插头、适配器）以及电器保护设备等。
２、特点：（1）兼容性（可适用于各种应用系统）；（2）开放性（符合多种国际上现行的标准，因此几乎对所有著名厂商的产品都是开放的，并对所有通信协议也是支持的）；（3）灵活性；（4）可靠性；（5）先进性（采用光纤与双绞线混合布线方式。所有布线均采用最新国际标准，链路均按照八芯双绞线配置；5类双绞线带宽可达100MHz,6类双绞线带宽可达200MHz）；（6）经济性
3、综合布线系统的组成：工作区子系统，配线（水平）子系统，干线（垂直）子系统，设备间子系统，管理子系统，建筑群子系统
4、系统标准：（1）ANSI／TIA／EIA 568-A；（2）TIA/EIA 568-B.1 TIA/EIA 568-B.2 TIA/EIA 568-B.3;(3)ISO/IEC 11801;（4）GB/T 50311-2000 GB/T 50312-2000；
三、以太网组网的基本方法
1、以太网物理层标准的命名方法：IEEE 802.3 x Type-y name(其中x为数据传输速度，Type表示传输方式基带（BASE）或者频带，y表示网段的最大长度或者选材，name表示局域网的名
称)
2、10BASE-T标准以太网的设计方法：
（1）局域网的LLC子网层采用802.2标准
（2）局域网的MAC层采用CSMA/CD方法
（3）物理结构取决于它选用的物理层标准，如：A、10BASE-5使用粗同轴电缆；B、10BASE-10采用细同轴电缆；C、10BASE-T采用非屏蔽双绞线、集线器（Hub）、RJ-45接口
3、组网方法按集线器使用方式分类：
A、单一集线器结构：从结点到集线器非屏蔽双绞线的最大长度为100m;
B、多集线器结构:局域网中的所有结点都在一个冲突域中
C、堆叠式集线器结构：由一个基础集线器与多个扩展集线器组成；基础集线器是具有网络管理功能的独立集线器；局域网中的所有结点都在一个冲突域中
4、快速以太网组网的设计方法（支持全双工工作模式的拓扑结构为星型）
（1）基本特点：在保持传统的以太网帧结构与介质访问控制方法不变的基础上将数据传输速率从10Mbps提供到100Mbps;其协议标准为802.3u；支持全双工和半双工两种工作模式。
（2）介质专用接口MII：将物理层与MAC子层分离开来。使物理层将速率从10Mbps提升到100Mbps时，传输介质和信号编码方式的变化不会影响MAC子层。向上通过与MAC层的接口提供载波侦听信号和冲突检测信号，向下支持10Mbps和100Mbps速率接口以及与集线器交换控制信息的功能
（3）物理层标准：
A、100BASE-TX：两对5类非屏蔽双绞线（UTP），一对用于发送，一对用于接收，最大长度为100m;数据传输采用4B/5B编码方法；采用全双工方式工作；
B、100BASE-T4：四对3类非屏蔽双绞线，三对用于传输数据，一对用于冲突检测，最大长度为100m；数据传输采用8B/6T编码方法；采用半双工方式工作；
C、100BASE-FX：两对光纤，一对用于发送，一对用于接收，最大长度为415m;数据传输采用4B/5B-NRZI编码方式；采用全双工方式工作
5、10Mbps和100Mbps速率自动化协商功能
（1）自动确定非屏蔽双绞线远端连接的设备使用的是半双工（CSMA/CD）的10Mbps的工作模式，还是全双工的100Mpbs的工作模式
（2）向其他结点发布连接设备的工作模式
（3）与远端连接设备交换工作模式相关参数，协调和确定双方的工作模式
（4）自动协商功能自动选择共有的最高性能的工作模式
（5）限制条件：只能用于使用双绞线的以太网，且规定自动协商过程需在500ms内完成
（6）根据工作模式性能，这些协议优先级从高到低为：100BASE-TX或者100BASE-FX全双工模式>100BASE—T4>100BASE-TX半双工模式>100BASE-T全双工模式>100BASE-T半双工模式
四、局域网互联设备类型
1、中继器（repeater）
（1）作用：真假以太网中同轴电缆的长度，将衰弱和变形后的信号将，经过接收、放大、整形的工作过程使得信号的波形和幅度达到协议规定的要求，然后再向与它相连的另一电缆发送出去
（2）中继器工作在物理层：
A、只能起到对传输介质上的波形的接收、放大整形与转发的作用，这是物理层的功能
B、其工作不涉及真的结构，不对帧的内容作任何处理。中继器只能起到增加传输介质长度的作用。中继器连接的几个缆段属于同一个局域网，连接在不同缆段上的多个结点都共享一个冲突域。中继器不属于网络互联设备。
（3）一个局域网中使用中继器连接doge缆段的数量是有限的（10BASE-5标准中，只能有四个中继器，连接3个缆段，网络中两结点的最大距离为2800m）
2、集线器
（1）工作在物理层：所有结点通过双绞线连接到一个集线器上，当一个结点发送数据时，所有结点都能接收到数据，因此集线器工作在物理层，所有连接在一个集线器的结点都属于同一冲突域
（2）有多个断不可，每个端口通过一个RJ-45接口与网卡连接
（3）以太网是典型的总线型结构，当它的物理层采用10BASE-T协议时，所有结点都通过双绞线连接到一个集线器上，它们任执行CSMA/CD
介质访问控制方法。但从外部结构来看，其物理结构是星型的。
3、网桥（在数据链路层实训网络互联的设备）
（1）特征：A、能互联两个采用不同数据链路层协议、不同传输介质与不同传输速率的网络；以接收、转换与地址过滤的方式实现互联网网络之间的通信
（2）实现：
（3）分类：
A、由帧转发策略分为透明网桥（802.1d）和源路由网桥
B、由端口数分为多端口网桥和双端口网桥
C、由连接线路分为普通局域网网桥、无线网桥与远程网桥
（4）网桥最重要的工作是构建和维护MAC地址表（记录不同结点的物理地址和网桥转发端口关系）
（5）透明网桥：
A、由每个网桥自己来进行路由选择，网桥对于互联局域网网络的结点是透明的。
B、一般用于两个MAC子层相同的网段之间的互联
C、易安装，即插即用设备
D、其MAC地址表要记录３类信息：站地址、端口和时间
4、交换机
与网桥的区别：
（1）在组成结构上：网桥通常是在一台典型的个人计算机上配置而成，其内部结构一般只有一个CPU，通过软件方法完成网桥的接收、存储、地址过滤与转发等功能。其地址过滤速度和转发速率等性能受到结构的限制；交换机使用为帧转发设计的专用集成芯片ASIC或采用多个CPU并发工作的计算结构。

（3）端口数量：网桥最多24个，交换机最多128个。

三级网络技术备考笔记知识点整理(4)

一、分组转发（在互联网中路由器转发IP地址分组的物理传输过程与数据报转发机制）
1、分类：直接转发、间接转发
2、直接转发：源主机与目的主机在同一个网络
3、间接转发：源主机与目的主机不在同一个网络
二、路由选择算法
1、主要参数：
（1）跳数（传输过程中经过路由器的个数）；（2）带宽（链路的传播速度）；（3）延时；（4）负载（单位时间内路由器或线路的通信量）；（5）可靠性（传输过程中的误码率）；（6）开销（传输过程中的耗费）
2、理想的路由选择算法的特点：（1）正确、稳定、公正；（2）尽量简单；（3）必须能够适应网络拓扑和通信量的变化；（4）最佳的（以最低的开销转发分组）
3、表驱动的路由选择算法：
（1）静态路由选择算法：简单且开销较小，但不能及时适应网络状态的变化。其中路由器为静态路由表，由人工方式创建，无法自动更新
（2）动态路由选择算法：能较的适应网络状态的变化，但实现较复杂，开销也较大。器路由表为动态路由表。
三、自治系统（AS）
1、特点：有权决定本自治系统中具体使用那种路由协议，核心是路由寻址的“自治”，其表现为：自治系统内部的路由器了解内部网络的全部路由信息并能够通过一条路径将发送到其他自治系统的分组转发本系统的主干道上；自治系统内部的路由器要向主干路由器报告内部路由信息
2、路由选择协议的分类：内部网关协议（IGP）和外部网关协议（EGP）
3、内部网关协议（IGP）是自治系统内部使用的路由选择协议，主要有：
（1）路由信息协议（RIP）
（2）开放最短路径优先（OSPF）协议
A、使用分布式链路状态；
B、要求路由器发送的信息是本路由器与哪些路由器相邻以及状态信息的度量（费用、距离、开销、延时等）；
C、要求当链路状态发生变化的时候用洪泛法向所有路由器发送此信息；
D、所有的路由器最终都能建立一个链路状态数据库，实际上是全网的拓扑结构图，并且在全网范围内是保持一致的；
E、将自治系统划分成多个更小的范围叫做区域，每个区域有一个32位的标识符（点分十进制），在一个区域的路由器不能超过200个
4、外部网关协议（EGP）是在个自治系统之间使用的路由选择协议，主要有边界网关协议（BGP），其四种分组为：open(打开分组，用来与相邻的BGP发言人建立连接)，keepalive（保活分组，上一状态为open，用来确认打开报文，以及周期性地证实相邻边界路由器的存在），update（更新分组，核心，发送某一路由的的信息，以及列出要撤销的多条路由），notification（通知分组，发送检测到的差错）

第三章


一、标准分类的IP地址
　１、定义：由网络好和主机号组成，长度为32位，用总分十进制表示。常用A、B、C类IP地址采用包括“网络号+主机号”的两级的层次结构(RFC1812)，其中：A类地址，网络号长度为7位，允许分配A类地址的网络只有128个；B类地址，网络号长度为14位，允许分配B类地址的网络号16384个。
　２、关于A类地址：网络号第一位为0，其余7位可分配，但网络号为0和127的留作特殊用途，网络号位10的用作专用地址
3、关于B类地址：网络号前两位为10，可分配网络号长度为14
4、关于C类地址：网络号前三位为110，可分配网络号长度为21
5、特殊地址形式：
A、直接广播地址：主机号全为1的地址，用来使路由器将一个分组以广播方式发送给特定网络上的所有主机
B、受限广播地址：32位全为1的地址（255.255.255.255），用来将一个分组以广播方式发送给本网络的所有主机，路由器则阻挡该分组通过，将广播功能限制在本网络内部
C、“这个网络上的特定主机”地址：
D、回送地址：127.0.0.0，用于网络软件测试和本地进程间通信
TCP/IP协议规定规定，含网络号为127的分组不能出现在任何网络上。主机和路由器不能为该地址广播任何寻址信息。

“Ping”应用程序可以发送一个回送地址作为目的地址的分组，以测试IP软件能否接受和发送分组。

一个进程可以使用回送地址来发送一个分组给本机的另一个进程，用来测试本地进程之间的通信。
二、划分子网的IP地址
1、定义：在标准的IP地址的基础上，增加子网号的三级地址结构，即：“网络号-子网号-主机号”的三级结构
2、子网的概念：允许将网络划分成多个部分供内部使用，但对外部来说仍像一个网络一样。
3、划分子网技术的要点：
（1）三级层次的网络结构：net ID－Subnet ID——host ID
（2）同一个子网中的子网号必须相同
（3）可用于A、B或C类的任意一类的IP地址
（4）子网间的距离必须很近
（5）分配子网是一个单位内部的事，不需要向ICANN申请和改变任何外部的数据库
（6）在Internet文献中，一个子网也称作一个IP网络或一个网络
三、构成超网的无类域间路由(CIDR)技术
1、出现原因：希望解决Internet扩展中存在的两大问题（地址有效利用率问题和路由器的工作效率问题），将现有的IP地址合并成较大的具有更多主机地址的路由域
2、技术特点：
（1）用“网络前缀”代替“网络号+子网号”，形成新的无分类的二级地址结果，即地址表示为<网络前缀，主机号>
（2）CIDR将网络前缀相同的连续的IP地址组成一个“CIDR地址块”

四、网络地址转换（NAT）技术：为每个公司分配一个或少量的IP地址，用于传输Internet流量。在公司内部的每台主机上分配一个不能在Internet上使用的保留的专用IP地址，用于公司内部网络的通信，如需访问外部Internet主机，必须运行网络地址转换（NAT）的主机或路由器将内部专用IP地址转换成全局IP地址。
