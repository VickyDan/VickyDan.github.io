三级网络技术备考知识点整理(7)

一、路由器工作原理
1、基本功能
（1）路由选择：依据目的IP地址的网络地址，通过路由选择算法确定一条从源结点到目的结点的最佳路由
A、核心：下一跳路由器的IP地址（路由表）
B、实现方法：使用路由选择算法，根据实际网络连接情况和网络性能，建立网络拓扑结构图，以建立路由选择和转发的基础。同时路由选择算法根据各自的判断原则（带宽、时延、负载、路由器跳数等因素），为网络上的路由产生一个权值。一般来说，权值愈小，路由愈佳。然后将最佳路由信息保存在一个路由表中。当网络拓扑发生变化时，路由选择协议会重新计算最佳路由，并更新路由表
（2）分组转发：按照路由选择所指出的路由将数据分组从源结点转发到目的结点
A、实现方法：接收到一个数据分组时，查看分组头中的目的IP地址的网络部分查询路由表。如果表中给出的是到达目的网络地址的下一跳路由器的IP地址，则按路由表给出的路径将数据分组转发到下一跳路由器，由下一跳路由器继续转发，直到将数据分组转发到目的端；如果目的网络与路由器的一个端口直接相连，那么对应于目的网络地址的路由表表项给出的是目的端口，这种情况下，路由器将数据分组直接发送给目的端口；如果路由表中既没有下一跳路由地址又没有目的端口时，将数据分组转发给缺省路由，由该路由所连的路由器继续转发到目的端，如果仍没有找到该分组的网络的路由信息，则将该分组丢弃。
2、路由表
（1）基本结构：有目的网络地址及其所对应的目的端口或下一跳路由地址和缺省路由
（2）管理距离：衡量路由器中给定的路由信息源的“可信度”。管理距离越小，可信度越高。
（3）权值：由带宽、时延、负载、跳数、可靠性、滴答数、花费等决定，权值愈小，路由愈佳
eg:[110/22]——>该条路由信息源的管理距离为110，权值为22

1、CPU：Motorola68030或Orion/R4600
2、内存：分类有只读内存（ROM）、随机存储器（RAM）、非易失性随机存储器（NVRAM）、
闪存（Flash）
3、接口
4、存储器
三、工作模式：用户模式、特权模式、设置模式、全局配置模式、其他配置模式、RXBOOT模式（维护模式）
四、路由器的基本操作与配置方法
1、配置方式
（1）使用控制端口配置
（2）使用AUX端口连接一台modem，通过拨号远程配置路由器
（3）使用telnet远程登录到路由器上配置路由器
（4）使用TFTP服务，以拷贝配置文件、修改配置文件的方式配置路由器
（5）通过网络管理协议SNMP修改路由器配置文件的方式
2、使用控制端口配置：与用控制端口配置交换机基本类似
3、使用telnet命令远程登录到路由器上配置路由器
（1）必备条件：
A、作为模拟终端的计算机和路由器都必须连网，它们能够相互通信
B、计算机必须拥有访问路由器的权限
C、路由器必须预先配置好远程登录密码
（2）配置（只需配置远程登录密码即可）
全局配置模式下：
Router（config）#line vty 0 15
Router(config-line)#password 7 222307
(3)具体步骤：在“运行”窗口输入“telnet <路由器IP地址>”
4、使用TFTP服务配置路由器
（1）必备条件：在网络上配置一台计算机作为TFTP服务器，TFTP服务器上必须安装并运行TFTP Server软件
（2）操作步骤
A、拷贝配置文件到TFTP服务器：特权用户模式下，write或copy命名
B、从TFTP服务器拷贝配置文件到路由器
五、路由器的基本配置及公用命令
1、基本配置
(1)配置路由器主机名：Router（config）#hostname
(2)配置超级用户口令（密码）：
Router（config）#secret
Router(config)#password
(3)设置系统时钟：Router#calendar set hh:mm:ss <1~31> MONTH <1993~2035>
2、公用命令
（1）退出命令：exit(逐级退出)；end（直接退出到特权用户模式）
（2）保存配置：
Router#write memory//保存到路由器NVRAM中
Router#write tftp//保存到TFTP服务器
（3）删除配置（全部）：Router＃write erase
（4）网络的基本测试命令
A、talnet
B、ping
C、trace
D、show
六、路由器的接口配置（在全局配置模式下，通过interface <接口号进入接口配置模式）
1、基本配置
（1）配置接口描述信息：Router(config-if)#description (一般描述连接到哪儿)
（2）配置宽带信息：Router(config-if)#bandwidth 2500000(2.5Gbps)
（3）配置接口的IP地址：Router(config-if)#ip address <子网掩码>
（4）接口的开启与关闭：shutdown(关闭)、no shutdown(接口配置模式)
2、局域网接口配置
（1）配置标准以太网接口（Ethernet，e）
（2）配置快速以太网接口(FastEthernet,f)
（3）配置千兆以太网接口（GigabitEthernet,g）
3、广域网串口配置
（1）配置异步串行接口（Asyns,a,主要用于连接Modem设备，为用户提供拨号上网服务）
（2）配置高速同步串行接口（Serial，s,主要用于DDN专线、帧中继、卫星、微波等广域网连接）（PPP或HDLC）
（3）配置POS接口（链路层协议PPP和HDLC）
4、loopback接口的配置（虚拟接口），主要设置IP address
七、静态路由配置（仅适合小型的，网络拓扑没有变化的局域网和采用点到点连接的网络互连环境）
方法：全局配置模式下，ip route <目的网络地址><子网掩码><下一跳路由器的IP地址>
删除：no ip route
八、动态路由协议的配置
1、RIP的配置（限制最大跳数为15，不支持可变掩码）
（1）基本配置
Router(config)#router rip//进入RIP路由协议的配置模式
Router(config-route)#network <目的地址>
（2）RIP的高级配置
A、配置被动接口：在RIP路由配置模式下，使用“passive-interface”指定一个路由接口为被动接口；eg:Router(config-route)#passive-interface ethernet 0//接口号
B、配置路由过滤（distribute-list）：在指定的路由器接口上，过滤接收/发送的路由更新信息
Router(config-route)#distribute-list 12 in ethernet 0
C、配置管理距离（distance）(1~255)：Router（config-route）#distance 50
D、定义邻居路由器(neighbor)：Router(config-route)#neighbor
2、OSPF动态路由协议的配置（区域ID为0或者0.0.0.0为骨干区域）
（1）基本配置
A、进入OSPF配置模式：Router(config)#router ospf <进程号>
B、用network命令定义参与OSPF的网络地址：
　　Router(config-route)#network ip <子网号> <子网掩码的反码> area <区域号>
C、定义某一特定范围子网的聚合：Router(config-route)#area <区域号> range <子网地址> <子网掩码>
（2）几个常用OSPF参数的配置
A、配置被动接口：passive-interface
B、配置路由过滤：distribute-list
C、配置管理距离：distance(默认为110)
D、配置引入外部路由到OSPF的参数
a、引入外部路由的花费值：redistribute metric
b、引入外部路由时缺省的标记：redistribute tag
c、引入外部路由时缺省的外部路由类型：
类型2：redistribute connected metric-type
九、路由器的DHCP功能及其配置
1、动态地址分配：使用动态主机配置协议(DHCP)，由网络站点提出DHCP请求，从DHCP服务器上自动获取一个IP地址与缺省网关、域名、域名服务器等相关的TCP/IP属性的信息。在网络站点退出登录后，就将地址自动释放（动态分配的IP地址是一个临时的、不固定的地址）
2、DHCP的工作原理
采用客户机/服务器（C/S）工作模式
DHCP服务器的两个功能：（1）建立和分配IP地址池；(2)接收并处理DHCP客户端提出的DHCP请求（分配一个可用的IP地址，并将相应的信息反馈给客户。客户端系统关闭后自动收回IP地址）
DHCP客户端的功能：提交DHCP请求，请求DHCP服务器给它分配一个IP地址并接收和使用DHCP临时配给它的IP地址进行网络连接
DHCP协议允许网络上配置多台DHCP服务器。当DHCP客户端收到多个应答信息时，通常会挑选最先到达的信息；同时DHCP客户与它选中的DHCP服务器获取IP地址及其他的TCP／IP属性信息。
DHCP协议允许使用备份DHCP服务器的功能
3、DHCP客户端从DHCP服务器申请一个IP地址的基本操作步骤：
主机A DHCPDISCOVER(boradcast） DHCP Server
DHCPOFFER(unicast)
DHCPREQUEST(boradcast)
DHCPACK(unicast)
4、路由器作为DHCP服务器的配置
（1）配置IP地址池的名称，并进入DHCP　Pool配置模式
　　　Router(config)#ip dhcp pool
（2）配置IP地址的IP子网地址和子网掩码：
Router(dhcp-config)#<网络号>[子网掩码]
配置不参与动态分配的IP地址（除外地）
Router(dhcp-config)#ip dhcp excluded-address low-address[high-address]
（3）配置IP地址池的缺省网关（网关地址最多可配置8个，但一般只需配一个）
Router(dhcp-config)#default-router address[address1 … address7]
（4）配置IP地址池的域名系统（用于为DHCP客户指定一个域名和域名服务器）
A、配置IP地址池的域名：Router(dhcp-config)#domain-name
B、配置IP地址池的域名服务器的IP地址：
Router(dhcp-config)#dns-server address[address1 … address7]
（5）配置IP地址池的地址租用时间（缺省为１天）
　Router(dhcp-config)#lease {days [hours] [munites]|infinite}//infinite为无限时间
（6）取消地址冲突记录日志
Router(dhcp-config)#no ip dhcp onflict logging
十、路由器IP访问控制列表的功能及其配置
1、访问控制列表：用于提供初级防火墙功能；告诉路由器哪些数据包可以接受，哪些需要拒绝；是一系列有网络管理员预先配置好的，允许和拒绝匹配规则的集合；其核心是根据特定规则进行数据包过滤，对网络进行安全控制，防止病毒包、扫描包、攻击包等通过路由器去攻击网络，保证网络的安全性
2、IP访问控制列表功能简介
（1）主要功能：包过滤
（2）过滤准则：基于源地址、目的地址、各种协议和端口号的过滤准则
（3）分类：
A、标准访问控制列表：只能检查数据包的源地址，根据源网络、子网络或主机IP地址来决定对包的过滤；表号为1~99，扩展表号1300~1999
B、扩展访问控制列表：可以检查数据包的源地址、目的地址、指定的协议和端口号来对数据包进行过滤；表号为100~199，扩展表号2000~2699
3、配置IP访问控制列表：一个连续列表，至少由一个“permit”和一个或多个“deny”语句组成；可以用名字或表号来标识和引用
（1）用access-list命令配置访问控制列表
A、配置标准访问控制列表
Router(config)#access-list {permit|deny} <源地址> <子网掩码的反码>
B、配置扩展访问控制列表
Router(config)#access-list {permit|deny} protocal source wildcard-mask destination wildcard-mask [operator] [operand]//operatar:操作符，“lt”(小于)，”gt”(大于)，”eq”(等于)，”neq”(不等于);operand:端口号
（2）ip access-list命令配置访问控制列表（配置标准和扩展的方法基本相同）
A、进入标准(扩展)访问控制模式：
Router(config)#ip access-list standard(extended) number(name)
B、标准(扩展)访问控制模式下：
permit(deny) protocal source wildcard-mask distination wildcard-mask [operator][opercand]
